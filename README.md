# 🎯 MIFB — Million Items Filtering

список из 1,000,000 элементов с выбором, сортировкой Drag & Drop, инфинити-скроллом и batch-очередью.

## API Endpoints

### Основные
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/api/left?search=&offset=0&limit=20` | Левый список (невыбранные) |
| `GET` | `/api/right?search=&offset=0&limit=20` | Правый список (выбранные) |
| `POST` | `/api/select` | Выбрать элемент `{ "id": 123 }` |
| `POST` | `/api/deselect` | Отменить выбор `{ "id": 123 }` |
| `POST` | `/api/reorder` | Изменить порядок `{ "order": [1,2,3] }` |
| `POST` | `/api/items` | Добавить новый элемент `{ "id": 1000001 }` |
| `POST` | `/api/batch` | Пакетная обработка действий |

### Служебные
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/api/health` | Состояние сервера |
| `GET` | `/api/version` | Версия и хеш коммита |

## Технологии

### Бэкенд
- **Express.js** + TypeScript
- Хранение 1M элементов в памяти (Map)
- Ручной CORS (никаких конфликтов с helmet)

### Фронтенд
- **React 18** + TypeScript + Vite
- **TanStack Query** — кэширование, инфинити-скролл
- **TanStack Virtual** — виртуализация (только видимые элементы в DOM)
- **dnd-kit** — Drag & Drop сортировка
- **Монорепозиторий** — front и back в одном репозитории

## Реализованный функционал

### Левое окно
- Отображение всех элементов, кроме выбранных
- Фильтрация по ID (с debounce 300мс)
- Инфинити-скролл (подгрузка по 20 элементов)
- Добавление новых элементов с валидацией ID > 1,000,000

### Правое окно
- Отображение выбранных элементов
- Фильтрация по ID (независимая от левого списка)
- Drag & Drop сортировка (с оптимистичным обновлением)
- Инфинити-скролл
- Кнопка возврата в левый список

### Общее
- Состояние хранится на сервере (общее для всех пользователей)
- Batch-очередь с дедупликацией (отправка раз в 1 сек)
- Оптимистичные обновления UI
- Версионирование в футере (front/back/commit/env)
- Полная поддержка мобильных устройств (через PointerSensor)

## Особенности реализации

### Batch-очередь с дедупликацией
- Действия собираются в очередь
- При 3+ действиях — отправка мгновенная
- При 1-2 действиях — таймаут 2 секунды
- Дедупликация: select + deselect для одного ID сокращаются

### Оптимистичные обновления
- Клик → элемент исчезает из UI мгновенно
- При ошибке сервера — элемент возвращается
- Инвалидация кэша после успешной отправки

### Drag & Drop
- Работает с отфильтрованным списком
- Сохраняет полный порядок (fullOrder)
- Визуальный overlay при перетаскивании

## Производительность

- **Память бэкенда**: ~50 MB на 1,000,000 элементов
- **Сеть**: 1 batch-запрос вместо 10 отдельных
- **Рендеринг**: только видимые элементы (виртуализация)
- **Кэширование**: TanStack Query хранит загруженные страницы
